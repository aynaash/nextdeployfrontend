# Quick Start

Get your Next.js application deployed in under 5 minutes with NextDeploy. This guide assumes you have NextDeploy CLI installed and a Next.js project ready to deploy.

## Prerequisites Checklist

Before we begin, make sure you have:

- ‚úÖ **NextDeploy CLI installed** ([Installation Guide](/docs/installation))
- ‚úÖ **A Next.js application** (create one with `npx create-next-app@latest`)
- ‚úÖ **A deployment target** (VPS, cloud server, or local Docker)
- ‚úÖ **SSH access** to your server (for VPS deployments)

<Info>
Don't have a Next.js app? Create one quickly with: `npx create-next-app@latest my-app --typescript --tailwind --eslint --app --src-dir`
</Info>

## Step 1: Initialize NextDeploy

Navigate to your Next.js project and initialize NextDeploy:

<CodeBlock title="Initialize NextDeploy in your project" language="bash">
{`# Navigate to your Next.js project
cd my-nextjs-app

# Initialize NextDeploy
nextdeploy init

# This creates:
# - nextdeploy.config.js (main configuration)
# - .nextdeploy/ (local cache and logs)
# - Updates .gitignore`}
</CodeBlock>

The initialization wizard will ask you a few questions:

<CodeBlock title="Initialization wizard" language="text">
{`? What's your application name? my-nextjs-app
? What framework are you using? Next.js
? What's your preferred deployment strategy? Blue-Green
? Enable monitoring? Yes
? Enable automatic backups? Yes

‚úÖ NextDeploy initialized successfully!`}
</CodeBlock>

## Step 3: Add Health Check Endpoint

Create a health check endpoint for NextDeploy to monitor your application:

<CodeBlock title="src/app/api/health/route.ts" language="typescript">
{`import { NextResponse } from 'next/server'

export async function GET() {
  try {
    // Basic health check
    const health = {
      status: 'healthy',
      timestamp: new Date().toISOString(),
      uptime: process.uptime(),
      version: process.env.npm_package_version || '1.0.0'
    }
    
    return NextResponse.json(health)
  } catch (error) {
    return NextResponse.json(
      { status: 'unhealthy', error: 'Health check failed' },
      { status: 503 }
    )
  }
}`}
</CodeBlock>

## Common Deployment Patterns

### Environment Variables

<CodeBlock title="Set environment variables" language="bash">
{`# Set environment variables
nextdeploy env set production DATABASE_URL "postgresql://..."
nextdeploy env set production API_KEY "your-api-key"

# Or use environment files
nextdeploy env load production .env.production

# View current environment
nextdeploy env list production`}
</CodeBlock>

### Custom Build Commands

<CodeBlock title="Custom build configuration" language="javascript">
{`// nextdeploy.config.js
module.exports = {
  app: {
    name: 'my-nextjs-app',
    framework: 'nextjs',
    buildCommand: 'npm run build:production',
    startCommand: 'npm run start:production',
    installCommand: 'npm ci --only=production'
  },
  build: {
    env: {
      NODE_ENV: 'production',
      NEXT_TELEMETRY_DISABLED: '1'
    },
    hooks: {
      preBuild: ['npm run lint', 'npm run test'],
      postBuild: ['npm run analyze']
    }
  }
}`}
</CodeBlock>

### Database Integration

<CodeBlock title="Database configuration" language="javascript">
{`// nextdeploy.config.js
module.exports = {
  // ... other config
  database: {
    type: 'postgresql',
    host: process.env.DATABASE_HOST,
    migrations: {
      enabled: true,
      path: './migrations'
    }
  },
  hooks: {
    beforeDeploy: ['npm run db:migrate'],
    afterDeploy: ['npm run db:seed']
  }
}`}
</CodeBlock>

## Troubleshooting Quick Fixes

### Deployment Fails

<CodeBlock title="Debug deployment issues" language="bash">
{`# Check detailed logs
nextdeploy logs production --level debug

# Validate configuration again
nextdeploy validate --verbose

# Test connection manually
ssh deploy@your-server.com

# Check server resources
nextdeploy system production`}
</CodeBlock>

### Application Won't Start

<CodeBlock title="Debug application startup" language="bash">
{`# Check application logs
nextdeploy logs production --app

# Test health endpoint
nextdeploy health production

# Check port availability
nextdeploy exec production "netstat -tlnp | grep :3000"

# Restart application
nextdeploy restart production`}
</CodeBlock>

### Build Issues

<CodeBlock title="Debug build problems" language="bash">
{`# Check build logs
nextdeploy logs production --build

# Test build locally
npm run build

# Clear build cache
nextdeploy cache clear production

# Rebuild from scratch
nextdeploy deploy production --no-cache`}
</CodeBlock>
